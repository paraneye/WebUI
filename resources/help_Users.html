Users

Users list

All generated Sigma users are listed. Total number of users is displayed with the number of available user licenses.

Adding New User

Selecting Add New displays pop up window for adding a new user. Fields denoted with '*' are required fields.

  • User Name: login ID of the user
  • Employee ID: employee ID of the user
  • First Name: first name of the user
  • Last Name: last name of the user
  • Phone Number: phone number of the user
  • Email Address: email address of the user
  • Company: company the user works for. If you cannot find the company, you can add a company to ¡°Companies¡± under Global Settings
  • Photo: photo image of the user. You should choose an image file smaller than 200Kbyte
  • Project and Role: project that the user takes part in. If there are projects and roles available, you can choose projects and roles for the user. Having multiple projects and roles are allowable.

  • Creating username will send welcome email to user¡¯s email address with username, temporary password, and URL of log in webpage in it.

User Authentication

Users are authenticated by username and password. Forgot password on login page gives user password reset email with temporary password in it.

Editing User

Selecting User Name from the users list pops up Edit window. Administrator only can edit user¡¯s information.

  • All fields except User Name are editable.

Deleting User

Administrator only can delete user name by selecting Delete button on Edit window.

Import File

Administrator can add bulk users by importing excel file. Before starting to import users information, you need to download template file and fill it out with user information. You can download a template file from Import File popup window. In the template file, column titles denoted with '*' are required fields. Notes in Readme sheet will help when you fill the template out.

  • Clicking Browse button opens pop-up windows asking for choosing a excel file from local computer.
  • To see what information is needed, download a excel template file. Fields denoted with '*' in it are required fields.
  • Selecting Save button imports users¡¯ data from the excel file and then shows the result.

Import History

After importing is done, import results and logs are available. You can see which item failed for which reason. Downloading the list of failures as an excel file can help make corrections on those failures and then import it again.

  • Clicking Download shows import failures and fail reasons. You can download failures as a form of excel file.
  • To delete import logs, put a check to import logs you want to delete, and select Delete button.
