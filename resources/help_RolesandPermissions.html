Roles and Permissions

Roles list

All created roles are listed with description. Administrator can create roles for Sigma users and grant access permissions to Sigma service menus. For user¡¯s convenience, roles required for construction project management are available by default.

Adding New Role

Selecting Add New moves to Add New Role page. Administrator only can add new role and assign access permissions

  • Name : role name
  • Description : description of the role
  • Copy Permission from : select one of existing roles to copy the permission from

Assigning Permissions

For the role you selected or are generating, you can grant access permissions by putting a check to each permission name. There are four groups of permissions, Global Settings, Project, TrueTask, and Crew.

  • Global Settings
    • Manager companies : user can add, edit, and delete companies
    • View companies : user can view companies
    • Manage library : user can add, edit, and delete library
    • View library : user can view library
    • Manage drawing type : user can add, edit, and delete drawing type
    • View drawing type : user can view drawing type
    • Manage progress type : user can add, edit, and delete progress type
    • View progress type : user can view progress type
    • Manage task category and type : user can add, edit, and delete task category and type
    • View task category and type : user can view task category and type
    • Manage cost code : user can add, edit, and delete cost code
    • View cost code : user can view cost code
    • Manage workflows : user can edit workflow information
    • View workflows : user can view workflow information
  • Project
    • Manage drawings : user can add, edit, and delete drawings
    • View drawings : user can view drawings
    • Manage schedule : user can add and edit schedule
    • View schedule : user can view schedule
    • Manage MTO : user can perform material takeoff
    • View component : user can view component
    • Manage members : user can add, edit, and delete members
    • View members : user can view members
    • Manage resources : user can add, edit, and delete human resources
    • View resources : user can view human resources
    • Manage information : user can edit project information
    • View information : user can view project information
    • Manage documents : user can add, edit, and delete documents
    • View documents : user can view documents
    • Manage form : user can add, edit, and delete forms
    • View form : user can view forms
    • View reports and records : user can view reports and records
    • Drawing viewer : user can use drawing viewer
    • IWP viewer : user can use IWP viewer
  • TrueTask
    • Build schedule : user can create, edit, and delete a work package by schedule line item
    • Build IWP : user can create, edit, and delete IWPs
    • Update IWP schedule : user can change start and finish date of IWP
    • View IWP schedule : user can view start and finish date of IWP
    • Manage manpower loading : user can change the amount of crew resources to load for IWP
    • View manpower loading : user can view the amount of crew resources loaded for IWP
    • Assemble IWP : user can generate IWP binder by collecting all required information
    • View signoff status : user can view signoff status of workflows
    • Drawing viewer : user can use drawing viewer
    • IWP viewer : user can use IWP viewer
  • Crew
    • Brass in : user can use the functionality to check crew members in
    • Brass out : user can use the functionality to check crew members out
    • Manage time and progress : user can use the functionality to record crew members¡¯ daily timesheet and report daily work progress
    • Manage ITR : user can use the functionality to perform inspection and test fieldwork and record result
    • View signoff status : user can view signoff status of workflows
    • Drawing viewer : user can use drawing viewer
    • IWP viewer : user can use IWP viewer

Editing Role and Permissions

Selecting Role Name from the roles list moves to Edit Role page. Administrator only can change role information and permissions setting. Roles available by default are not allowed to change.

Deleting Role

Administrator only can delete roles. Put a check to role you want to delete and select Delete on the roles list screen.

Permission Report

This shows overall matrix of permission assignment for all roles. You can change permission settings for roles.
