Human Resources

Human Resources List

All employees assigned to the project are listed. Human resources are the employees who participate in a project but have no Sigma user ID. Generally they do their own work offline so that they don¡¯t need to access Sigma website. They are, for example, crew members who do field work, field support, field staff, client representatives, or guest.

Adding New Resource

Selecting Add New displays pop-up window for adding a new employee. Fields denoted with '*' are required fields.

  • Employee ID : employee ID of the employee
  • First Name : first name of the employee
  • Last Name : last name of the employee
  • Phone Number : phone number of the employee
  • Email Address : email address of the employee
  • Role : select a role among Crew, Support, Field staff, Client, and Guest.
  • NFC Card ID : ID of NFC card that Crew member has. If you select Crew for Role, this data is required.
  • PIN : PIN for NFC card. If you select Crew for Role, this data is required.
  • Company : company the employee works for. If you cannot find the company, you can add a company to ¡°Companies¡± under Global Settings
  • Photo : photo image of the employee. You should choose an image file smaller than 200Kbyte

Editing Resource

Selecting Employee ID from the employee list pops up Edit window. All fields are editable.

Deleting Resource

Put a check to the employee you want to delete and select Delete on the employees list screen.

Import File

You can add bulk users by importing excel file. Before starting to import employee information, you need to download template file and fill it out with employee information. You can download a template file from Import File popup window. In the template file, column titles denoted with '*' are required fields. Notes in Readme sheet will help when you fill the template out.

  • Clicking Browse button opens pop-up windows asking for choosing a excel file from local computer.
  • To see what information is needed, download a excel template file. Fields denoted with '*' in it are required fields.
  • Selecting Save button imports employees¡¯ data from the excel file and then shows the result.

Import History

After importing is done, import results and logs are available. You can see which item failed for which reason. Downloading the list of failures as an excel file can help make corrections on those failures and then import it again.

  • Clicking Download shows import failures and fail reasons. You can download failures as a form of excel file.
  • To delete import logs, put a check to import logs you want to delete, and select Delete button.
