User Profile

When you click your username that appears on the navigation bar, user profile window pops up. It displays your information and allows you to change default project and reset password.

  • Setting Default Project : if you set default project, project overview screen of the default project instead of home screen opens next time you log in. If you have only one project assigned, default project is automatically set to it.
  • Changing Password : reset your password. Password is case-sensitive.

Global Settings

Global Settings allows administrator to establish database that is globally used for all projects. When a new project is set up project manager needs to pick specific values out of this database. Global settings¡¯ database consists of followings,

  • Users who get Sigma login ID
  • Roles that take part in projects and get assigned to Sigma users
  • Access permission to service menus for roles
  • Workflows to be used for defined work process
  • Companies that are site-owner, contractor, and sub-contractors
  • Libraries of materials, equipment, and consumables
  • Drawing types
  • Task categories and types that are defined to classify materials in library
  • Progress types that are defined task progress milestones and weights to apply progress milestones
  • Cost Codes to define the types of cost

Selecting Project

Once a project is created and project manager assigns Sigma users to project members for it, users get to access the project by selecting the project name from project name list box. Administrator can create a new project by selecting Add New Project.

Projects

Projects that user is assigned to as a project member are listed. User can move to the project overview page by selecting a project name from project list. Administrator can create a new project by selecting Add New Project and close the project by selecting Close button

Messages

Messages sent to the user are listed with read button and delete button.
